# Full-Stack Developer role

Please find the technical challenge below. It’s designed to take 3–4 hours and focuses purely on practical skills.

You are expected to deliver the solution within 2–3 calendar days. Please confirm receipt of this email and let me know your availability to start the test so I can follow up accordingly.

> [!WARNING] You may consult online resources to support your solution. However, please ensure that your work reflects your own understanding and approach, and is not generated by AI tools.

## What you’ll build

A small, containerized web app to manage vehicle repair estimates with:

A Python (FastAPI) REST API (login, list estimates with status filter, create estimate, update status).

A simple React UI (login, table listing, create form).

Docker Compose to run everything locally.

You can complete the core scope and optionally do bonus items for extra credit.

## How to run the project (candidate steps)

Create a project folder on your machine and open it in your terminal.

Inside the folder, create the required files, including:

.env.sample (with the necessary environment variables)
.env (you can copy from .env.sample)
docker-compose.yml
Any other files needed to run the application
Once your setup is ready, run the application using Docker:
docker-compose up --build

Your application should expose:

API docs at: <http://localhost:8000/docs>

Frontend at: <http://localhost:5173>

If Docker is not available, you may run the backend and frontend locally. Please include clear instructions on how to do so. However, using Docker is preferred.

## What to deliver

A public GitHub link to your solution (or a ZIP if you prefer).

Include:

Your code for backend and frontend.

docker-compose.yml and any config you made.

At least one backend test (e.g., using pytest).

A brief AWS deployment outline in the README (5–8 bullets: how you’d deploy using ECS/Lambda, S3+CloudFront, RDS, IAM, secrets, CI/CD).

## Evaluation (what we look for)

API correctness, validation, and clean error handling.

A working React UI that uses the API (login, list + filter, create).

Clear, organized code and a short README with run steps.

Docker setup that works on a clean machine.

(Optional bonus) Pagination, simple rate-limiting, migrations, React Query, CI, healthcheck.
